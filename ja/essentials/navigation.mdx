ナビゲーションメニューは、すべてのウェブサイトにあるリンクのリストです。

新しいページを追加するたびに`docs.json`を更新することになるでしょう。ページは自動的には表示されません。

## ナビゲーション構文

私たちのナビゲーション構文は再帰的であり、ネストしたナビゲーショングループを作成できます。ページ名に`.mdx`を含める必要はありません。

<CodeGroup>

```json 通常のナビゲーション
"navigation": {
  "tabs": [
    {
      "tab": "ドキュメント",
      "groups": [
        {
          "group": "はじめに",
          "pages": ["quickstart"]
        }
      ]
    }
  ]
}
```

```json ネストされたナビゲーション
"navigation": {
  "tabs": [
    {
      "tab": "ドキュメント",
      "groups": [
        {
          "group": "はじめに",
          "pages": [
            "quickstart",
            {
              "group": "ネストされたリファレンスページ",
              "pages": ["nested-reference-page"]
            }
          ]
        }
      ]
    }
  ]
}
```

</CodeGroup>

## フォルダー

MDXファイルをフォルダーに入れ、`docs.json`内のパスを更新してください。

たとえば、`https://yoursite.com/your-folder/your-page`の位置にページを持つには、`your-folder`というフォルダーを作成し、その中に`your-page.mdx`というMDXファイルを含めます。

<Warning>

フォルダーの名前に`api`を使用することはできませんが、別のフォルダーの中に入れれば使用可能です。MintlifyはNext.jsを使用しており、トップレベルの`api`フォルダーは内部サーバーコールのために予約されています。`api-reference`のようなフォルダー名は受け入れられます。

</Warning>

```json フォルダー付きのナビゲーション
"navigation": {
  "tabs": [
    {
      "tab": "ドキュメント",
      "groups": [
        {
          "group": "グループ名",
          "pages": ["your-folder/your-page"]
        }
      ]
    }
  ]
}
```

## 非表示ページ

`docs.json`に含まれていないMDXファイルはサイドバーには表示されませんが、検索バーを通じてアクセスしたり、直接リンクを貼ったりすることができます。
