<Info>
  **先决条件**:
  - Node.js 版本 19 或更高
  - 包含 `docs.json` 文件的文档仓库
</Info>

请按照以下步骤在您的操作系统上安装和运行 Mintlify。

<Steps>
<Step title="安装 Mintlify CLI">

```bash
npm i -g mint
```
</Step>

<Step title="本地预览">

导航到您的 `docs.json` 文件所在的文档目录，并运行以下命令：

```bash
mint dev
```

您的文档的本地预览将在 `http://localhost:3000` 上可用。

</Step>
</Steps>

## 自定义端口

默认情况下，Mintlify 使用端口 3000。您可以使用 `--port` 标志自定义 Mintlify 运行的端口。例如，要在端口 3333 上运行 Mintlify，请使用此命令：

```bash
mint dev --port 3333
```

如果您尝试在已在使用的端口上运行 Mintlify，它将使用下一个可用端口：

```md
端口 3000 已在使用中。正在尝试 3001。
```

## Mintlify 版本

请注意，每个 CLI 版本都与特定版本的 Mintlify 相关联。如果您的本地预览与生产版本不一致，请更新 CLI：

```bash
npm mint update
```

## 验证链接

CLI 可以帮助验证您文档中的链接。要识别任何损坏的链接，请使用以下命令：

```bash
mint broken-links
```

## 代码格式化

我们建议在您的 IDE 上使用扩展程序来识别和格式化 MDX。如果您是 VSCode 用户，可以考虑使用 [MDX VSCode 扩展](https://marketplace.visualstudio.com/items?itemName=unifiedjs.vscode-mdx) 进行语法高亮，并使用 [Prettier](https://marketplace.visualstudio.com/items?itemName=esbenp.prettier-vscode) 进行代码格式化。

## 故障排除

<AccordionGroup>
  <Accordion title='错误：无法在使用 darwin-arm64 运行时的环境中加载 "sharp" 模块'>

    这可能是由于 node 版本过时所致。请尝试以下操作：
    1. 移除当前安装的 CLI 版本：`npm remove -g mint`
    2. 升级到 Node v19 或更高版本。
    3. 重新安装 CLI：`npm i -g mint`
  </Accordion>

  <Accordion title="问题：遇到未知错误">
  
    解决方案：转到您设备的根目录并删除 `~/.mintlify` 文件夹。然后再次运行 `mint dev`。
  </Accordion>
</AccordionGroup>

想知道最新的 CLI 版本中有什么变化吗？请查看 [CLI 更新日志](https://www.npmjs.com/package/mintlify?activeTab=versions)。
