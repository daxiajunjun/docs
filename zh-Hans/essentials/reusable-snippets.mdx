## 创建自定义代码片段

**先决条件**：您必须在 `snippets` 目录中创建您的代码片段文件。

<Note>
  `snippets` 目录中的任何页面都将被视为代码片段，并且不会被渲染为独立页面。如果您想从代码片段创建独立页面，请将代码片段导入另一个文件并将其作为组件调用。
</Note>

### 默认导出

1. 将您希望在多个位置重用的内容添加到您的代码片段文件中。或者，您可以添加变量，这些变量可以在您导入代码片段时通过 props 填充。

```mdx snippets/my-snippet.mdx
你好世界！这是我希望在多个页面中重用的内容。我今天的关键词是 {word}。
```

<Warning>
  您希望重用的内容必须位于 `snippets` 目录中，以便导入正常工作。
</Warning>

2. 将代码片段导入您的目标文件。

```mdx destination-file.mdx
---
title: 我的标题
description: 我的描述
---

import MySnippet from '/snippets/path/to/my-snippet.mdx';

## 标题

Lorem impsum dolor sit amet.

<MySnippet word="香蕉" />
```

### 可重用变量

1. 从您的代码片段文件中导出一个变量：

```mdx snippets/path/to/custom-variables.mdx
export const myName = '我的名字';

export const myObject = { fruit: '草莓' };
```

2. 从您的目标文件中导入代码片段并使用该变量：

```mdx destination-file.mdx
---
title: 我的标题
description: 我的描述
---

import { myName, myObject } from '/snippets/path/to/custom-variables.mdx';

你好，我的名字是 {myName}，我喜欢 {myObject.fruit}。
```

### 可重用组件

1. 在您的代码片段文件中，通过以箭头函数的形式导出组件来创建一个接受 props 的组件。

```mdx snippets/custom-component.mdx
export const MyComponent = ({ title }) => (
  <div>
    <h1>{title}</h1>
    <p>... 代码片段内容 ...</p>
  </div>
);
```

<Warning>
  MDX 不会在箭头函数的主体内编译。尽可能坚持使用 HTML 语法，或者如果需要使用 MDX，请使用默认导出。
</Warning>

2. 将代码片段导入您的目标文件并传入 props

```mdx destination-file.mdx
---
title: 我的标题
description: 我的描述
---

import { MyComponent } from '/snippets/custom-component.mdx';

Lorem ipsum dolor sit amet.

<MyComponent title={'自定义标题'} />
```
